# 신살(神殺) 분석 카드 Design Spec

## Goal

사주 결과 페이지의 오행 균형 조언 카드 바로 아래에 "사주에 해당하는 대표 신살" 카드를 추가한다.
도화살·역마살·화개살·천을귀인·양인살 5개를 판별해 해당하는 것만 리스트로 표시하며, API 호출 없이 정적 규칙 기반으로 즉시 렌더링한다.

## UX 결정사항

- **범위**: 대표 신살 5개 (도화살, 역마살, 화개살, 천을귀인, 양인살)
- **판별 기준**: 일지(日支) 기준 삼합법 (도화/역마/화개), 일간(日干) 기준 (천을귀인/양인살) — 통용되는 단순화 방식
- **표시**: 발견된 신살만 리스트로 표시. 하나도 없으면 카드는 유지하되 "특별한 신살이 없는 균형 잡힌 사주예요" 문구 표시
- **배치**: `OhaengAdvice` 카드 바로 아래, 동일 카드 컨테이너 스타일 (`bg-card rounded-2xl p-4`)
- **데이터**: 정적 규칙 기반 (AI 호출 없음)

### `lib/sinsal.ts`

```ts
import type { SajuResult } from './saju-calculator';

export type SinsalType = '도화살' | '역마살' | '화개살' | '천을귀인' | '양인살';

export interface SinsalEntry {
  hanja: string;
  description: string; // 한 줄 해석
}

export const SINSAL_INFO: Record<SinsalType, SinsalEntry>
export function getSinsals(saju: SajuResult): SinsalType[]
```

**삼합군 판별 테이블 (`SAMHAP_GROUP`):**

지지 12개를 4개 삼합군으로 매핑한다.

| 지지 | 삼합군 |
|------|--------|
| 寅, 午, 戌 | 寅午戌 |
| 申, 子, 辰 | 申子辰 |
| 巳, 酉, 丑 | 巳酉丑 |
| 亥, 卯, 未 | 亥卯未 |

**삼합군별 신살 지지 테이블:**

| 삼합군 | 도화살 지지 | 역마살 지지 | 화개살 지지 |
|--------|------------|------------|------------|
| 寅午戌 | 卯 | 申 | 戌 |
| 申子辰 | 酉 | 寅 | 辰 |
| 巳酉丑 | 午 | 亥 | 丑 |
| 亥卯未 | 子 | 巳 | 未 |

**천을귀인 테이블 (일간 기준):**

| 일간 | 귀인 지지 |
|------|----------|
| 甲, 戊, 庚 | 丑, 未 |
| 乙, 己 | 子, 申 |
| 丙, 丁 | 亥, 酉 |
| 辛 | 午, 寅 |
| 壬, 癸 | 卯, 巳 |

**양인살 테이블 (양간만 해당 — 甲丙戊庚壬):**

| 일간 | 양인 지지 |
|------|----------|
| 甲 | 卯 |
| 丙 | 午 |
| 戊 | 午 |
| 庚 | 酉 |
| 壬 | 子 |

음간(乙丁己辛癸)은 양인살 없음.

**`getSinsals` 로직:**

1. 사주 4개 기둥(년/월/일/시, 시는 `null`일 수 있음)의 지지를 모아 `branches: string[]` 생성
2. 일지(`saju.day.ji`)로 삼합군을 조회. 그룹이 정해지면:
   - 도화 지지가 `branches`에 있으면 `'도화살'` 추가
   - 역마 지지가 `branches`에 있으면 `'역마살'` 추가
   - 화개 지지가 `branches`에 있으면 `'화개살'` 추가
3. 일간(`saju.ilgan`)으로 천을귀인 지지 목록을 조회. 목록 중 하나라도 `branches`에 있으면 `'천을귀인'` 추가
4. 일간이 양간이면 양인 지지를 조회. `branches`에 있으면 `'양인살'` 추가
5. 발견된 신살 배열 반환 (순서: 도화살 → 역마살 → 화개살 → 천을귀인 → 양인살), 없으면 빈 배열

**5개 신살 해석 데이터 (`SINSAL_INFO`):**

| 신살 | 한자 | 해석 |
|------|------|------|
| 도화살 | 桃花殺 | 매력과 인기운이 강해요. 사람을 끄는 힘이 있지만 감정 기복에 유의하세요 |
| 역마살 | 驛馬殺 | 이동과 변화의 기운이 강해요. 여행, 이주, 직업 변동이 잦을 수 있어요 |
| 화개살 | 華蓋殺 | 예술·종교·학문에 대한 깊은 통찰력이 있어요. 고독을 즐기는 성향도 있어요 |
| 천을귀인 | 天乙貴人 | 주변에서 도움을 주는 사람이 많아요. 어려운 순간에 귀인의 조력을 받기 쉬워요 |
| 양인살 | 羊刃殺 | 강한 추진력과 결단력을 가졌어요. 직설적인 표현으로 마찰이 생기지 않게 조심하세요 |

### `components/SinsalCard.tsx`

```tsx
interface SinsalCardProps {
  saju: SajuResult;
}
```

- `getSinsals(saju)`로 해당 신살 목록 계산
- 헤더: `✨ 신살(神殺) 분석`
- 목록이 비어있으면: "특별한 신살이 없는 균형 잡힌 사주예요" 문구만 표시 (카드는 항상 렌더링됨 — `OhaengAdvice`와 달리 `null` 반환하지 않음)
- 목록이 있으면: 각 신살을 `{이름}({한자})` + 한 줄 해석으로 리스트 렌더링
- 카드 스타일: `bg-card rounded-2xl p-4` (기존 패턴)
- `memo()` 적용

### `SajuResultContent.tsx` 수정

`<OhaengAdvice ohaeng={result.ohaeng} />` 바로 아래에 삽입:

```tsx
import SinsalCard from '@/components/SinsalCard';
// ...
<OhaengAdvice ohaeng={result.ohaeng} />

```

## 테스트

`lib/__tests__/sinsal.test.ts`에서 `getSinsals` 순수 함수 테스트:

- 도화살이 성립하는 사주 (일지 삼합군 + 도화 지지 존재)
- 역마살이 성립하는 사주
- 화개살이 성립하는 사주
- 천을귀인이 성립하는 사주 (일간 기준)
- 양인살이 성립하는 사주 (양간 기준)
- 음간 일간은 양인살 후보에서 제외됨
- 아무 신살도 없는 사주 → 빈 배열
- 여러 신살이 동시에 성립하는 사주 → 모두 반환, 정해진 순서로

`SinsalCard.tsx`는 컴포넌트 테스트 인프라가 없으므로 테스트 생략 (기존 `OhaengAdvice` 패턴과 동일).

## 비고

- API 호출 없음 — 즉시 렌더링
- 신살 판별은 명리학파에 따라 기준(년지 vs 일지)과 세부 지지 목록이 다를 수 있음 — 이 구현은 일지 기준 삼합법 + 통용되는 단순화 표를 사용
- `OhaengAdvice`와 달리 신살이 없어도 카드 자체는 항상 표시 (긍정적 메시지로 빈 상태 처리)
